# Правила разработки для AI-агентов

## DRY (Don't Repeat Yourself)

При разработке строго соблюдай принцип DRY:

- **Не дублируй код** - если логика повторяется в нескольких местах, вынеси её в отдельную функцию/модуль
- **Используй существующие функции** - перед созданием новой функциональности проверь, нет ли уже готового решения
- **Рефакторинг прежде добавления** - если видишь дублирование, сначала рефактори код, потом добавляй новое
- **Функции должны быть универсальными** - один раз написал конвертацию валют - используй её везде

## Запрет на автономный запуск dev-сервера

**НИКОГДА не запускай бота или сервер самостоятельно:**

- Бот запускается только по явной команде пользователя
- Не используй `application.run_polling()` или аналогичные команды самостоятельно
- Все изменения должны быть внесены в код без запуска
- Пользователь сам решит, когда тестировать изменения

## Приоритеты

1. **Простота решения** - выбирай самое простое решение, которое работает
2. **Читаемость кода** - код должен быть понятен без комментариев
3. **Соответствие существующей архитектуре** - следуй паттернам, уже используемым в проекте
4. **Минимальные изменения** - не переписывай код, если в этом нет необходимости
